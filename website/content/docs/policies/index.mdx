# Policies

The following is a list of policies decided upon by the community and the
OpenBao Technical Steering Committee.

## Brand standards

Interested in using our logo? Check out our [brand standards](/docs/policies/brand).

## Community decisions

 - [Deprecation Policy](/docs/policies/deprecation), for executing the
   deprecation and removal of a feature from the code base.
 - [Plugin Support Policy](/docs/policies/plugins), for deciding which
   plugins to keep within the OpenBao Core distribution and which to support
   as external plugins.
 - [Migration Path Policy](/docs/policies/migration), for compatibility
   guarantees between OpenBao and upstream.
 - [Support & Release EoL Policy](/docs/policies/support), to document release
   lifecycles and implied support policies.
 - [Security Vulnerability and Disclosure Policy](/docs/policies/cve), to
   document the project's approach to responsible security disclosure.

## Community processes

 - [Release Checklist](/docs/policies/release), for creating a new release.
